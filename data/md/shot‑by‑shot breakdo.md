## **Format for Python Script**

The above shot list is structured with clear fields that can be parsed by a Python script. For easy reading, you can store the data in a CSV or JSON file. A suggested JSON format:

```json
{
  "shots": [
    {
      "id": "01-01",
      "duration": 4,
      "camera": "Wide, static",
      "visual_beat": "Two kids (6‑8) leap into a massive mud puddle – pure joy",
      "animation_notes": "Exaggerated squash‑and‑stretch on jumps; anticipation before leap",
      "assets": ["Park trees", "mud puddle", "kids"],
      "lighting": "Golden hour, long shadows",
      "sound": "Wet squelch, giggles"
    },
    ...
  ]
}
```

The script will read this data and for each shot create:
- **Maya (.ma)**: a camera with the given name, maybe a locator representing the shot, and placeholder geometry for key assets (e.g., a cube for the puddle, simple characters).
- **Houdini (.hip)**: a camera node, a null for the shot, and basic geometry placeholders.

The exact implementation of the script can be tailored to the pipeline, but the shot breakdown above provides all the necessary information to automate the initial scene setup.
